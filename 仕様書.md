# マイル販売見積りWebアプリ 仕様書

## 1. 概要

マイルの販売において、顧客からの問い合わせに対して迅速に見積りを提示するためのWebアプリケーション。
従来のGoogleスプレッドシートでの管理をWebアプリに置き換え、レート変更を容易にする。

## 2. 背景・課題

- 現在はGoogleスプレッドシートで見積りを管理している
- レートが変わるたびにスプレッドシートの数式や値を手動で変更する必要があり面倒
- 2026年7月1日に価格改定が行われ、今後も状況に応じて改定予定

## 3. 対象デバイス

- Mac（ブラウザ）
- Windows（ブラウザ）
- iPhone（Safari / ブラウザ）

→ レスポンシブデザインのWebアプリとして構築する

## 4. 機能要件

### 4.1 見積り計算機能（メイン画面）

#### 入力
| 項目 | 説明 |
|------|------|
| マイル数 | 販売するマイル数を入力（300の倍数推奨、バリデーションあり） |

#### 価格帯（2つ）
| 名称 | 初期単価（円/マイル） | 説明 |
|------|----------------------|------|
| ヤフオク | 2.52 | ヤフオク出品時の価格 |
| 銀行振込 | 2.42 | 銀行振込時の価格 |

#### レート入力欄
- メイン画面上部にヤフオクレートと銀行振込レートの入力欄を常時表示
- 値を変更するだけで、即座に価格計算とメッセージ文面に反映される
- 変更したレートはローカルストレージに自動保存（次回アクセス時も維持）

#### 出力（自動計算）
- 各価格帯の合計金額（円換算は整数に四捨五入）
- 例：120,000マイルの場合
  - ヤフオク（2.52円/マイル）: 302,400円
  - 銀行振込（2.42円/マイル）: 290,400円

#### 入力バリデーション
- 300の倍数でない場合は警告を表示（計算自体は実行する）
- 数値以外の入力はエラー表示

### 4.2 メッセージ自動生成機能

見積り計算結果をもとに、顧客へ送信するメッセージを自動生成する。
レートや金額は入力値に連動してリアルタイムに文面が更新される。

#### メッセージテンプレート

```
この度はお問い合わせいただきありがとうございました。

300マイルごとのご提供となりますため、{マイル数}マイルでお見積りいたします。

ヤフオクですと、{ヤフオク単価}円/マイル（{ヤフオク金額}円）のところお値引きさせて頂き、{ヤフオク値引き金額}円（{ヤフオク値引き単価}円/マイル）でいかがでしょうか。よろしければ出品いたしますのでご都合をお知らせください。

最もお安く済ませたい場合、直接当方への銀行振込みを承ります。
こちらでよろしければ{銀行振込金額}円（{銀行振込単価}円/マイル）でご提供が可能です。よろしければ振込口座をご案内いたしますのでご都合をお知らせください。

以上、ご検討の程よろしくお願いします。
```

※ ヤフオク値引き金額 = ヤフオク金額を切りの良い数字に丸めた値（現行踏襲）

#### コピー機能
- 生成されたメッセージをワンタップ/クリックでクリップボードにコピー

## 5. 非機能要件

### 5.1 技術構成
- **フロントエンドのみ**のシングルページアプリケーション（SPA）
- サーバー不要（静的ホスティングで運用可能）
- HTML + CSS + JavaScript（フレームワーク不使用、単一HTMLファイル）

### 5.2 ホスティング
- GitHub Pages（無料、どのデバイスのブラウザからもアクセス可能）

### 5.3 レスポンシブデザイン
- iPhone SE～iPhone 16 Pro Maxに対応
- PC/タブレットでも使いやすいレイアウト
- ダークモード対応（OSの設定に追従）

### 5.4 パフォーマンス
- 初回ロード1秒以内
- 入力に対してリアルタイムで計算結果・メッセージを反映

### 5.5 データ保存
- ローカルストレージ使用（サーバーサイドDB不要）
- レート設定はブラウザごとに自動保存

├──────────────────────────────┤
│  ■ 送信メッセージ            │

- レートを変えるだけで価格もメッセージもリアルタイム更新
- iPhoneでもスクロールなしで主要情報が見渡せるコンパクト設計
